Hashes vs Arrays

Sunday, October 5th, 2014

This week I've been asked to discuss the differences between arrays and bases in ruby. Both arrays and hashes are very similar in that they both are used to store a "collection" or "series" of data. They can both grow and shrink in size and can hold any type of data, integers, floats, strings, etc. The difference between an array and hash is how the data is accessed. For example, arrays use a linear integer to index the data for storing or recalling a piece of data, while hashes use a "hash."" A hash can be a string, number or whatever. The differences are best explained with examples, the following is an example for an array of pets.

Pets = ["cat", "dog", "fish"]

In order to recall a specific piece of data from pets one would call Pets[n] where n represents a number, in this case, ranging from 0 to 2. Pets[0] would return "cat." Now for hashes, they are defined differently and an example with pet's could be used below where the pet of an owner could accessed.

Pets = {
'John' => "cat",
'Jack' => "dog",
'Jane' => "fish",
}

If you wanted to access an owner's pet one could utilize the hash, in this case, 'John' used with the hash, Pets['John'] to recall the data, "cat." Hashes are better used when the data being stored is not linear or ordered. This is my explanation of hashes and arrays. Thanks for reading.
